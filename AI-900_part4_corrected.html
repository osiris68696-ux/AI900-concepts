📋 負責任 AI 的原則 (Responsible AI Principles)

原則 挑戰或風險 範例
公平性 (Fairness) 偏差可能會影響結果。 貸款核准模型因為其訓練資料有偏差而在性別上的差別待遇。
可靠性與安全性 (Reliability & Safety) 錯誤可能造成傷害。 自動駕駛汽車遇到系統故障而導致車禍。
隱私權與安全性 (Privacy & Security) 私人資料可能遭到公開。 醫療診斷代理程式會使用敏感的患者資料進行訓練，而這些資料並未安全地儲存。
包容性 (Inclusiveness) 解決方案可能不適用於所有人。 預測性應用程式，無法為視障使用者提供音訊輸出。
透明度 (Transparency) 使用者必須信任複雜的系統。 以 AI 為基礎的財務工具會提出投資建議－這些工具以什麼為依據？
問責性 (Accountability) 誰應該對 AI 驅動的決策負責任？ 無辜的人因為來自臉部辨識的證據而被判罪－這是誰的責任？

🤖 機器學習的類型 (Types of Machine Learning)

監督式機器學習 (Supervised Learning)

定型資料包括已知標籤 (Labels are known)。

1. 迴歸 (Regression)

  • 標籤是數值 (Label is a number)。
  • 範例：根據日期、季節和天氣預測出售的冰淇淋數目。

2. 分類 (Classification)

  • 標籤是分類 (或稱 類別) (Label is a category)。
  • 二元分類 (Binary Classification): 標籤是/或不是類別 (例如：根據臨床資料預測患者是否具有罹患糖尿病的風險)。
  • 多類別分類 (Multiclass Classification): 標籤是多個類別之一 (例如：根據測量資料來預測企鵝的物種)。

非監督式機器學習 (Unsupervised Learning)

定型資料未標記 (Labels are unknown)。

叢集 (Clustering)

  • 類似的項目會分組在一起。
  • 範例：根據共同特性將植物物種分成群組。

📊 模型演算法和評估指標 (Algorithms & Metrics)

1. 迴歸 (Regression)

  • 範例：線性迴歸 (Linear Regression)
  • 評估指標 (Evaluation Metrics):
    • 平均絕對誤差 (MAE)
    • 平均平方誤差 (MSE)
    • 均方根誤差 (RMSE)
    • 決定係數 (R²)

2. 分類 (Classification)

  • 範例：羅吉斯迴歸 (Logistic Regression)
  • 評估指標 (每個類別和整體):
    • 準確度 (Accuracy)
    • 召回率 (Recall)
    • 精確度 (Precision)
    • F1 分數 (F1 Score)

3. 叢集 (Clustering)

  • 範例：K-Means
  • 評估指標:
    • 輪廓係數 (Silhouette) ⭐重點
    • 叢集內到形心的平均距離
    • 輪廓分數

🧠 深度學習 (Deep Learning)

人工神經網路 (Artificial Neural Networks)

  • 人類神經網路：神經元會對電化學刺激作出回應；觸發回應時，會將訊號傳遞至連接的神經元。
  • 每個神經元都是一個對輸入值 (x) 和權數 (w) 進行操作的函數。
  • 函數包在啟用函數 (Activation Function) 中，以判斷是否要將輸出傳遞到下一層。
  • 結構包含：輸入層 → 隱藏層 (Hidden Layers) → 輸出層。
損失 (Loss): 預測機率與實際標籤 (0 或 1) 之間的差異。在訓練過程中，會透過調整權重以盡量減少損失。

自動化機器學習 (Automated ML)

  • 可協助您執行機器學習訓練作業的逐步精靈。
  • 支援多種機器學習類型，包括迴歸、時間序列預測、分類、電腦視覺和自然語言處理工作。
  • 連結至您的資料、定義訓練作業和目標指標，以及部署最佳結果模型。

✨ 什麼是生成式 AI? (What is Generative AI?)

語言模型 (Language Model) 為核心
  • 自然語言產生 → 生成文字 (例如：為求職申請寫一封求職信)
  • 影像產生 → 生成影像 (例如：建立花店業務的標誌)
  • 程式碼產生 → 生成程式碼 (例如：撰寫 Python 程式碼以新增兩個數位)

語言模型 (Language Models) - 高階概觀

  • 訓練文件 → 編碼器 (Encoder) → 將文字標記編碼為向量 (內嵌/Embeddings)
  • 範例：
    • dog → [10, 3, 2]
    • cat → [10, 3, 1] (與 dog 相似)
    • puppy → [5, 2, 1]
    • skateboard → [3, 2, 1]
  • 內嵌向量會在多個維度中封裝語意屬性 (相關字有類似的方向)。
  • 輸入 (提示) → 解碼器 (Decoder) → 產生自然語言序列 → 輸出 (完成)

📐 語言模型: 大型和小型語言模型 (LLM vs SLM)

大型語言模型 (LLM) 小型語言模型 (SLM)
使用大量通用文字資料訓練 使用重點文字資料訓練
數十億個參數 較少的參數
在多種環境中提供全面的語言產生能力 專注於特定環境中的語言產生能力
大型尺寸可能會影響效能與便攜性 快速且可攜式
使用您自己的訓練資料微調耗時 (且昂貴) 更快速 (且成本較低) 以使用您自己的訓練資料微調
範例: Azure AI Foundry GPT 5 中的 Azure OpenAI, Mistral 7B, Meta Llama 3 範例: Microsoft Phi 4, Microsoft Orca 2, Hugging Face GPT Neo
